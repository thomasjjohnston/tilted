Tilted · Hand-to-Hand Transitions

Reducing Friction, Preserving Strategy

Core tenet: after acting, return to overview for strategic reassessment. But beta testers report friction finding the next pending hand. Six options for reducing that friction without auto-transitioning willy-nilly.

Baseline

Current Behavior

How it works today. Reference point.

Current flow: Tap pending hand → detail sheet opens → act → sheet dismisses back to the list at its current scroll position. User scrolls to find the next pending hand and taps it.
Friction: After acting, user lands on the list wherever it was scrolled. They have to scan and scroll to find the next pending hand.

Option 01

Auto-scroll to First Pending

After acting, the list auto-scrolls so the next pending hand is at the top.

Interaction: Tap pending hand → sheet opens → act → sheet dismisses. As it dismisses, the list smoothly scrolls so the next Action Required hand is at the top of the visible area. User sees it immediately without scrolling.
Trade-off: Subtle. Doesn't add new UI. May feel magical or may feel like the app "did something" unexpectedly.

Option 02

Sticky "Next Pending" Button

Persistent button at bottom of screen opens the next pending hand's detail sheet directly.

Interaction: A sticky button "Next: Hand 7 →" appears above the tab bar whenever there are pending hands. Tap it to directly open the next pending hand's detail sheet — no need to scroll the list or find the tile.
Trade-off: Extra UI chrome. But explicit, discoverable, and keeps the strategic overview optional (user can still scroll and pick manually).

Option 03

Swipe Gestures on Detail Sheet

Within the detail sheet, swipe left to jump to next pending, swipe right for previous. Stay in the sheet instead of returning to the list.

Interaction: Detail sheet is open. Swipe left to jump directly to the next pending hand's detail. Swipe right to go back. Swipe down (existing) still dismisses to list. Pagination dots at top show position.
Trade-off: Fast power-user flow — tap list once, then swipe through all pending. But bypasses the "always back to overview" principle. Could be offered alongside list-based flow, not as replacement.

Option 04

Pending Badge in Sheet Header

Detail sheet shows "2 more pending after this" with a pill badge. Clicking closes sheet and flows to list.

Interaction: When detail sheet is open, the header shows "Hand 3 · 2 more pending". After acting, sheet dismisses. List scrolls to next pending (combined with #01) OR user sees the pill fade and knows there's more.
Trade-off: Low impact, informational only. Best paired with auto-scroll (#01) or sticky button (#02).

Option 05

Combined: Sticky Button + Auto-scroll

The most aggressive friction reducer. Both behaviors combined.

Interaction: After acting, sheet dismisses AND list auto-scrolls to next pending. Sticky button "Next: Hand X →" persists at bottom. User can either (a) tap sticky button to open next directly, (b) see the auto-scrolled tile and tap it, or (c) ignore and scroll to pick a different pending.
Trade-off: Multiple paths to next. Strongest reduction of friction while preserving strategic overview. Adds both UI + motion.

Option 06

Combined: Swipe + Badge + "Back to List"

Power-user focused. Clear in-sheet navigation with explicit list return.

Interaction: Sheet has pagination dots at top, pending count badge, and a clear "All Hands ↑" button at top-left to return to list. Swipe left/right between pending hands. Swipe down or tap All Hands to overview.
Trade-off: Most explicit controls. No magic. User always knows how to navigate. Slightly more UI chrome.

Summary

Recommended combination: Option 05 (sticky button + auto-scroll) for the primary friction fix — low disruption to strategic overview, explicit fast path for "just advance." Consider Option 03 (swipe gestures) as a power-user enhancement on top.
