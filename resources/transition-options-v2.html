Tilted · Transitions v2 · Two-Button Pattern

Next Hand + Back to All Hands

Five options showing two button choice after acting. Both buttons visible so the user picks: auto-advance to next pending hand, or return to overview for strategic reassessment. Each option differs in where/how the buttons appear.

Option 01

Dual Footer (Permanent)

Both buttons always visible at the bottom of the detail sheet. After acting, action buttons hide and dual footer becomes the only interaction.

Interaction: Tap pending hand → sheet opens with action buttons (Fold/Call/etc) AND a dual footer (All Hands / Next). Take action → action buttons disappear, resolution shown above, dual footer remains. Tap All Hands to return to list, or Next Hand to open the next pending hand's sheet directly.

Option 02

Post-Action Modal

After acting, a small modal slides up with two large card-style options. Explicit "what now?" choice.

Interaction: Take action → resolution briefly shown → a compact modal slides up with two large cards: "Back to all hands" (shows how many pending remain) and "Next hand" (preview of the next pending hand). User taps one. The modal emphasizes "you made a choice" psychology.

Option 03

Inline Result + Dual CTA

After acting, the detail sheet content transforms into a result screen with dual buttons. Feels seamless — no modal.

Interaction: Take action → the detail sheet smoothly transitions: action buttons gone, result pushed up (cards + pot + winner), and two full-width buttons at bottom: ↑ All Hands (secondary) and Next Hand (primary, with preview of which hand). Tap one to navigate.

Option 05

Stacked CTA Column

Both buttons full-width, stacked vertically. "Next hand" is the primary with preview, "All hands" is a quieter secondary below.

Interaction: Same as Option 04 (full-screen result) but buttons stacked vertically instead of side-by-side. Primary button Next Hand is larger and richer (shows which hand, what's facing you). Secondary "Back to all hands" is a ghost-style button below. Emphasizes "Next" as the expected path while keeping "All hands" clearly available.
